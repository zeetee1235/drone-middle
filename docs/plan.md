# 제24회 한국로봇항공기경연대회 중급부문
## 1차 기술개발 계획서

---

## 1. 미션 환경

| 항목 | 내용 |
|------|------|
| Safe Area | 32 × 23m |
| Mission Area | 24 × 15m |
| 격자 크기 | 3 × 3m |
| 구획선 폭 | 10cm |
| ArUco 마커 수 | 4개 (위치 랜덤, 격자 교차점·꼭짓점에만 배치) |
| 비행 고도 | 2m |
| 버티포트 | 미션 구역 외부 / 지름 3m / 높이 0.7m |

---

## 2. 미션 수행 절차

```
[1] 이륙
    └─ 미션 구역 밖 버티포트에서 자율 이륙

[2] 격자 탐색 (Line Tracing)
    └─ 격자 선을 따라 자유 경로로 비행하며 ArUco 마커 탐색
    └─ 위에서 봤을 때 구획선 이탈 금지
    └─ 고도 2m 유지
    └─ 마커 발견 시 → 해당 위치에서 3초 호버링 → 탐색 계속

[3] 구조 경로 순회
    └─ 4개 마커 탐색 완료 후 ID 역순으로 정렬
    └─ 격자 라인 비행 불필요 (자유 경로)
    └─ 각 마커 재방문 시 3초 호버링 필수

[4] 귀환 및 착륙
    └─ 순회 완료 후 버티포트로 복귀
    └─ 자율 착륙
```

---

## 3. 평가 기준

| 평가 항목 | 세부 내용 |
|-----------|-----------|
| 위치 정확도 | 경로점 수평 x, y 오차 |
| 마커 인식 | ArUco 마커 식별 정확도 |
| 라인 트레이싱 | 고도 유지 정도, 구획선 이탈 정도 |
| 구조경로 추종 | 방문 순서 정확도, 경로점 위치 추종 성능 |
| 임무 수행 시간 | 전체 미션 완료까지 소요 시간 |
| **가산점** | 하드웨어 또는 소프트웨어 자체 개발 |

---

## 4. 전체 시스템 구성

```
┌─────────────────────────────────────────────┐
│               온보드 컴퓨터                   │
│  (Raspberry Pi 4 / Jetson Nano)              │
│  - ArUco 인식 모듈 (OpenCV)                  │
│  - 미션 매니저                                │
│  - ROS2 통신                                 │
└───────────────┬─────────────────────────────┘
                │ MAVLink
┌───────────────▼─────────────────────────────┐
│            비행 컨트롤러                      │
│       (Pixhawk / PX4 or ArduPilot)           │
│  - 자세 제어, 고도 유지                       │
│  - 하향 비전 + 광학 플로우 측위               │
└───────────────┬─────────────────────────────┘
                │
┌───────────────▼─────────────────────────────┐
│           기체 하드웨어                       │
│  - 쿼드로터 프레임                            │
│  - 모터 / ESC / 배터리                       │
│  - 하향 카메라 (ArUco 인식용)                │
└─────────────────────────────────────────────┘
                │ 무선통신
┌───────────────▼─────────────────────────────┐
│            지상국 (GCS)                       │
│         QGroundControl                       │
└─────────────────────────────────────────────┘
```

---

## 5. 소프트웨어 스택

| 역할 | 도구 |
|------|------|
| 비행 제어 | PX4 or ArduPilot |
| 미들웨어 | ROS2 |
| 영상처리 / 마커 인식 | OpenCV (ArUco 모듈) |
| 임무 관리 | 자체 개발 Mission Manager |
| 지상국 | QGroundControl |

---

## 6. 핵심 개발 항목

### 6-1. ArUco 마커 인식 모듈
- 하향 카메라 실시간 영상에서 마커 검출
- 마커 ID 확인 및 드론 기준 상대 위치 추정
- 오인식 방지를 위한 인식 확인 알고리즘 적용
- 최초 인식 및 재방문 시 자동 3초 호버링 트리거

### 6-2. Line Tracing 모듈
- GPS 비의존 3×3m 격자 좌표 맵 생성
- 격자 선 추종 자율비행 (x, y 오차 최소화)
- 고도 2m 자동 유지 제어
- 구획선 이탈 감지 및 보정

### 6-3. 미션 매니저
- **탐색 → 순회 → 귀환** 상태 전환 관리
- 발견된 마커 ID 리스트 수집 및 저장
- 역순 정렬 후 순회 경로 자동 생성
- 각 단계별 예외 처리 (마커 미인식 등)

### 6-4. 자율 이착륙
- 버티포트 좌표 기반 자동 이륙
- 귀환 후 정밀 자율 착륙
- 비전 보조 착륙 알고리즘 적용 검토

## 7. 개발 일정

| 단계 | 주요 내용 |
|------|-----------|
| 1단계 | 하드웨어 선정 및 프레임 조립, 비행 컨트롤러 기본 세팅 |
| 2단계 | 기본 자율비행 구현 (이착륙, GPS 웨이포인트) |
| 3단계 | ArUco 인식 모듈 개발 및 단독 테스트 |
| 4단계 | Line Tracing 모듈 개발 및 격자 비행 테스트 |
| 5단계 | 미션 매니저 통합 및 전체 시나리오 연동 |
| 6단계 | 실내 시뮬레이션 및 실외 비행 테스트 |
| 7단계 | 최종 튜닝 및 대회 준비 |

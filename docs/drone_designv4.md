# 경량 GPS 비의존 스프린트 드론 — 통합 설계 문서 v4 (최종)

**기반 규정**: 제24회 한국로봇항공기경연대회 중급부문  
**개발 방향**: Ingenuity Optical Flow 항법 / 교차점-to-교차점 탐색 / 구조 단계 완전 자유 레이싱 스프린트  
**작성일**: 2026-05-17

---

## 설계 철학

> 탐색 중에는 교차점만 찍고 지나간다.
> 마커 4개를 찾은 순간, 격자는 버리고 레이싱 드론이 된다.

이 문서의 최종 판단은 명확하다. **라인 PID는 구현하지 않는다.**  
교차점-to-교차점 경로 자체가 격자선 위를 지나가므로 구획선 이탈 점수는 자연스럽게 부분 획득하고, 개발 시간은 Optical Flow 정확도와 스프린트 속도 튜닝에 집중한다.

---

## 1. 비행 단계 전체 구조

```
Phase 0   이륙 + Home Pose 초기화
Phase 1   격자 탐색 (교차점 주파, 격자선 근처 비행)
            └ 마커 4개 발견 완료
Phase 2   구조 경로 (완전 자유 직선 스프린트, ID 역순)
Phase 3   귀환 + 비전 서보 착륙
```

**Phase 1과 Phase 2의 경계가 전략의 핵심이다.**  
Phase 1은 탐색 효율과 적당한 라인 점수를 위해 교차점 좌표만 따라가고,  
Phase 2부터는 격자를 완전히 무시하고 저장된 좌표 사이를 최단 직선으로 질주한다.

## 2. 점수 전략

### 2.1 항목별 목표

| 항목 | 배점 | 목표 | 전략 |
|------|------|------|------|
| 경로점 위치 추정 | 120점 | **상 등급** | 교차점 스냅으로 오차 0.5m 이내 |
| ArUco ID 식별 | 40점 | **전량 확보** | 고속 통과 중 인식 |
| 고도 유지 | 30점 | **상 등급** | ToF + PX4 튜닝 |
| 구획선 이탈 | 40점 | **부분 (15~25점)** | 교차점 직선 경로로 자연 획득 |
| 구조 방문 순서 | 40점 | **전량 확보** | 내림차순 정렬 로직 |
| 구조 통과 정확도 | 80점 | **상 등급** | 비전 재정렬 |
| 임무 수행 시간 | 100점 | **최상위권** | Phase 1 통과 인식 + Phase 2~3 스프린트 |
| 자체 개발 가산 | 50점 | **최대한** | Optical Flow 항법 어필 |
| **목표 합계** | **500점** | **≈ 430점** | |

### 2.2 핵심 판단

구획선 이탈 40점은 정성 평가라 최적화하기 어렵다. 교차점-to-교차점 비행만으로 격자선 근처를 자연스럽게 지나가므로 15~25점은 노릴 수 있다. 나머지 점수를 위해 라인 PID를 따로 구현하면 개발 시간이 Optical Flow, 고속 인식, 스프린트 튜닝에서 빠져나간다.

따라서 점수 전략의 우선순위는 다음과 같다.

```
1순위: 위치 추정 + ArUco ID + 구조 순서 + 구조 정확도
2순위: 임무 시간
3순위: 고도 유지
4순위: 구획선 이탈 부분 점수

명시적 제외: 라인 PID 기반 구획선 추종
```

## 3. 항법 기반 — Ingenuity Optical Flow

### 3.1 원리

화성 탐사 헬리콥터 Ingenuity는 GPS 없는 환경에서 하향 카메라로 지면을 연속 촬영하고 프레임 간 픽셀 이동량을 계산해 위치를 추정했다. 광학마우스가 표면을 읽어 이동량을 계산하는 것과 동일한 원리다.

```
광학마우스:  표면 촬영 → 픽셀 이동 → 커서 이동량
Ingenuity:  지면 촬영 → 픽셀 이동 → 드론 이동량
본 시스템:  지면 촬영 → 픽셀 이동 → 격자 좌표계 위치
```

### 3.2 센서 구성

| 센서 | 역할 | 특이사항 |
|------|------|----------|
| 하향 고정 카메라 | Optical Flow + ArUco + 교차점 검출 | 글로벌 셔터 필수 |
| IMU | 자세, 단기 가속도 적분 | 고주파 예측 |
| ToF 거리 센서 | 2m 고도 정밀 유지 | Optical Flow 스케일 보정 |
| 기압계 | 고도 보조 | FC 내장 |

### 3.3 위치 추정 흐름

```
IMU (수백 Hz)
  └→ 단기 자세 + 가속도 예측

Optical Flow (카메라 FPS)
  └→ 픽셀 이동량 계산
  └→ 고도 기반 실제 거리 환산
  └→ IMU 예측값 보정 (EKF 또는 상보 필터)

격자 교차점 인식 시
  └→ 현재 위치 = 격자 교차점 좌표 (강제 스냅, drift 리셋)

ArUco 마커 인식 시
  └→ 절대 위치 보정 + 마커 저장
```

### 3.4 교차점 스냅 — drift 억제 핵심

Optical Flow는 적분 방식이라 오차가 누적된다. 격자 간격 3m마다 교차점을 인식해 위치를 강제 정렬하면 누적 오차를 3m 단위로 리셋한다. 마커 발견 직후 저장하는 위치 좌표의 오차를 0.5m 이내로 유지하는 핵심 장치다.

```
교차점 인식
→ estimated_position = grid_node_coordinate
→ 이전 drift 폐기
→ Optical Flow 적분 재시작
```

## 5. Phase 1 — 격자 탐색

### 5.1 목표

교차점 54개를 빠짐없이 통과하며 ArUco 마커 4개를 발견한다.

### 5.2 탐색 경로 — Boustrophedon (S자 주파)

교차점만을 목표 지점으로 삼아 S자 패턴으로 이동한다. 격자선을 PID로 따라가지 않고, 교차점 좌표를 목표로 Optical Flow 위치 제어로 이동한다.

```
(0,0)→(3,0)→(6,0)→...→(24,0)     행 1, 9개 교차점
        ↓
(24,0)→(24,3)                      열 이동
        ↓
(24,3)→(21,3)→...→(0,3)           행 2, 9개 교차점
        ↓
...
(0,12)→(3,12)→...→(24,12)         행 5
        ↓
(24,12)→(24,15)
        ↓
(24,15)→(21,15)→...→(0,15)        행 6, 9개 교차점
```

교차점 간 이동은 직선이고, 이 직선들이 격자선과 거의 일치하므로 별도 PID 없이도 격자선 근처를 비행하게 된다. 여기서 라인은 "따라가는 대상"이 아니라 Optical Flow와 교차점 스냅을 보조하는 시각 기준이다.

### 5.3 속도

교차점 간 거리는 3m. 짧은 구간이라 삼각형 속도 프로파일을 사용한다. 목표는 일반 자율비행처럼 얌전히 이동하는 것이 아니라, 교차점 사이를 최대 가속과 최대 감속으로 찌르는 것이다.

```
가속 → 감속 (순항 없음)
최고 도달 속도 = √(a_max × 3m) 로 제한
```

초기 안전 목표는 4~6 m/s, 튜닝 목표는 6~8 m/s다. 가속도와 감속도는 카메라 블러, Optical Flow 오차, 기체 자세 안정 한계까지 밀어붙이며 실험으로 정한다.

열 이동 구간(3m 직각 전환)에서는 감속 → 방향 전환 → 재가속.

행 방향 연속 이동은 교차점마다 멈추지 않고 다음 교차점 좌표로 연속 목표 전환하여 속도를 최대한 유지한다. 교차점은 정지점이 아니라 위치 오차를 리셋하는 체크포인트다.

### 5.4 교차점 통과 중 ArUco 인식

정지하지 않고 이동하면서 하향 카메라로 교차점 위를 지나는 순간 ArUco를 인식한다. Phase 1에서는 마커 위에 멈추지 않는다. 발견한 ID와 좌표만 저장하고 그대로 다음 교차점으로 진행한다.

```
교차점 접근 1~2m 전:
  시야에 교차점 진입
  ArUco 검출 시작

교차점 통과 순간:
  연속 프레임 ArUco 검출 시도
  → 검출 성공 (N프레임 연속 동일 ID):
      교차점 스냅 실행
      ID + grid_node_coordinate 저장
      속도 유지 → 다음 교차점 진행
  → 검출 없음:
      교차점 스냅 실행
      다음 교차점으로 계속 가속

4개 발견 완료 시:
  현재 위치에서 즉시 Phase 2로 전환
  남은 교차점 탐색 중단
```

**왜 고속 통과 인식이 가능한가:**

2m 고도 + 120° 화각 카메라 기준 지면 폭 약 4.6m. 50cm 마커는 교차점 접근 약 2m 전부터 시야에 들어온다. 6 m/s 이동 시 마커가 시야에 머무는 시간 약 0.5초 = 60fps 기준 30프레임, 8 m/s에서도 약 22프레임이다. 정지 없이 ID를 확정하기에 충분한 프레임 수를 목표로 한다.

### 5.5 흔들린 프레임 처리

고속 이동 중 roll/pitch 변화가 큰 프레임은 ArUco 인식에서 제외한다. 속도는 줄이지 않는다.

```
IMU 자세 변화율 임계값 초과 프레임
→ 인식 건너뜀 (속도 유지)
→ 다음 안정 프레임에서 재시도
```

## 6. Phase 2 — 구조 경로 (완전 자유 스프린트)

### 6.1 규정 확인

규정 원문: "최적 구조 경로(역순)에 따라 경로점 이동 **구획선 추종 기능 미검증**"

격자선 추종 의무 없음. 완전 자유 이동.

### 6.2 방문 순서

저장된 마커 ID를 내림차순으로 정렬한다.

```
탐색 발견 예시: ID 3 → 1 → 4 → 2 (발견 순서 무관)
구조 방문 순서: ID 4 → 3 → 2 → 1
```

### 6.3 이동 전략

마커 간 이동을 완전 직선 스프린트로 수행한다. 격자 경로를 무시하고 유클리드 최단 거리로 이동한다. 이 구간이 실제 승부처다.

```
현재 위치 → 다음 마커 저장 좌표
  사다리꼴 속도 프로파일:
  가속 → 순항 → 감속
```

Phase 1보다 이동 거리가 짧고 방향 전환이 적어 실제 이동 시간이 크게 줄어드는 구간이다. 초기 안전 목표는 6~8 m/s, 튜닝 목표는 8~12 m/s다. 다만 마커 재검출 가능 거리와 제동 거리를 항상 먼저 계산한다.

```
필요 제동 거리 = v² / (2 × a_brake)
목표 접근 반경 = max(2m, 필요 제동 거리 + 인식 안정 거리)
```

### 6.4 마커 도착 시 위치 정확도 확보 (80점)

```
저장 좌표 2m 이내 진입
→ 감속
→ 하향 카메라 ArUco 재검출
→ 픽셀 오차 기반 수평 보정 (비전 서보)
    vx = kx * (image_cx - marker_cx)
    vy = ky * (image_cy - marker_cy)
→ 오차 0.5m 이내 + 속도 안정 확인
→ 3초 호버링 (채점 구간)
→ 다음 마커로 출발
```

## 7. Phase 3 — 귀환 및 착륙

### 7.1 귀환 경로

마지막 마커 호버링 완료 후 저장된 Home Pose 방향으로 직선 스프린트.

```
마지막 마커 좌표 (Grid 좌표계)
→ T_grid_home 변환
→ Home 예측 위치까지 직선 스프린트
→ 버티포트 반경 3m: 감속
→ 하향 카메라 버티포트 재검출
→ 비전 서보 중심 정렬
→ 자동 착륙
```

### 7.2 버티포트 비전 서보 착륙

```
버티포트 원형 외곽 또는 착륙 마커 검출
→ 중심 픽셀 오차 계산
→ 수평 보정 명령 생성
→ 조건 만족 시 단계적 하강:
    수평 오차 < 0.3m
    수평 속도 < 0.3m/s
    고도 안정
```

## 8. 미션 타임라인 추정

| 단계 | 거리 | 속도 | 이동 | 호버링 | 합계 |
|------|------|------|------|--------|------|
| Phase 0 이륙+초기화 | — | — | — | 3~5초 | 5초 |
| Phase 1 탐색 (54교차점) | ≈ 170m | 4~8 m/s | 22~42초 | 0초 | 22~42초 |
| Phase 2 구조 경로 | ≈ 20~50m | 6~12 m/s | 3~8초 | 12초 | 15~20초 |
| Phase 3 귀환+착륙 | ≈ 10~20m | 5~10 m/s | 2~4초 | 3~5초 | 5~9초 |
| **전체 예상** | | | | | **47~76초** |

Phase 1 탐색 중 마커 호버링을 제거하면 12초가 바로 줄어든다. 남는 고정 호버링은 구조 재방문 12초와 착륙 안정화뿐이다.  
Phase 2~3 자유 스프린트 구간은 경쟁 팀 대비 가장 큰 시간 이득을 내는 구간이며, Phase 1도 교차점 정지 없이 밀어붙여야 한다.

## 9. 미션 매니저 상태 기계

```
INIT
  └→ TAKEOFF
       └→ HOME_POSE_INIT
            └→ [Phase 1] INTERSECTION_SPRINT
                  ├→ PASS_THROUGH_MARKER_SAVE (발견 시, 정지 없음)
                  │    └→ INTERSECTION_SPRINT 계속
                  └→ [4개 완료] RESCUE_PLAN (내림차순 정렬)
                        └→ [Phase 2] RESCUE_SPRINT
                              └→ RESCUE_APPROACH (2m 이내)
                                   └→ VISION_REALIGN
                                        └→ RESCUE_HOVER (3초)
                                             └→ RESCUE_SPRINT (다음)
                                                  └→ [완료] RETURN_SPRINT
                                                        └→ [Phase 3] VERTIPORT_REACQUIRE
                                                              └→ VISION_SERVO_LAND
                                                                   └→ DONE
```

**예외 처리:**

| 상황 | 대응 |
|------|------|
| 교차점 통과 중 ArUco 미검출 | 스냅만 실행, 다음 교차점 진행 |
| 마커 후보 통과 후 미확정 | 탐색 재개 후 해당 교차점 재방문 큐 추가 |
| 위치 drift 과다 | 속도 제한 후 가장 가까운 교차점 재검출 |
| 구조 경로 마커 재검출 실패 | 저장 좌표 주변 소형 탐색 패턴 |
| 버티포트 재검출 실패 | Home Pose 주변 나선형 탐색 |
| 배터리 부족 | 즉시 귀환 모드 |

## 10. 하드웨어 구성

### 10.1 기체 선정 기준

| 기준 | 요구 사항 |
|------|-----------|
| 제어 | PX4 Offboard 또는 ArduPilot Guided |
| 탑재 여유 | 추가 장비 100g 이내 |
| 하향 장착 | 기체 하부 카메라 장착 가능 |
| 진동 | 영상에 심한 젤로 없을 것 |

### 10.2 추가 하드웨어 목록

| 구성품 | 목표 무게 | 핵심 요구사항 |
|--------|-----------|---------------|
| 하향 카메라 | 5~20g | **글로벌 셔터**, 60fps 이상, 화각 90~120° |
| 온보드 컴퓨터 | 25~70g | Raspberry Pi 4/5 또는 Jetson Orin Nano |
| 전원 모듈 (5V DC-DC) | 10~25g | 급가속 시 전압 강하 방지 |
| ToF 거리 센서 | 5~15g | 2m 고도 정밀 유지, Optical Flow 스케일 보정 |
| 통신 모듈 | 5~20g | 모니터링 + 비상 개입 |
| 카메라 마운트 | 10~30g | 진동 절연, 하향 고정 |
| **합계 목표** | **< 100g** | |

**글로벌 셔터가 중요한 이유:**  
롤링 셔터는 고속 이동 시 격자선과 ArUco가 기울어져 보이는 왜곡이 발생한다. Optical Flow 정확도와 ArUco 인식률 모두에 직접 영향을 준다. 스프린트 비행에서 글로벌 셔터는 선택이 아니라 필수다.

## 15. 제출 문서용 통합 문단

본 팀은 화성 탐사 헬리콥터 Ingenuity의 GPS 비의존 광학 항법을 핵심 기술로 채택한다. 광학마우스와 같은 원리로 하향 카메라가 지면을 연속 촬영하고 프레임 간 픽셀 이동량을 계산해 GPS 없이 실시간 위치를 추정하며, 격자 교차점을 인식할 때마다 누적 오차를 리셋하는 교차점 스냅 방식으로 위치 정확도 0.5m 이내를 유지한다.

탐색 단계에서는 라인 PID를 사용하지 않고 격자 교차점 54개를 목표로 S자 패턴을 고속 주파하며, 정지 없이 통과 중 ArUco 마커를 즉시 인식한다. 마커 4개 발견이 완료되는 순간 자유 비행 모드로 즉시 전환되어, 구조 경로와 귀환 구간에서는 격자 구조를 무시하고 저장된 마커 좌표 간 직선 스프린트를 수행한다. 각 마커 도착 직전에는 비전 서보 재정렬로 위치 정확도를 확보하고, 버티포트 귀환 시에는 하향 카메라 기반 중심 정렬로 자동 착륙한다.
